# SOP5 人物/场景设定清单与候选图池设计

## 1. 背景

SOP5 的目标是把已确认的完整剧本推进到生产前置资产确认：标准剧本、详细文字分镜、人物设定图、场景设定图。当前补充设计只覆盖人物/场景设定图部分，作为 `2026-06-29-sop5-script-storyboard-setup-design.md` 的细化。

核心问题：

- AI 抽取的人物/场景可能漏掉、抽错或包含不需要生成的路人群众。
- 当前单张卡片固定四个图片槽，不适合持续补图。
- 生成提示词不可见，用户无法判断图片是按什么逻辑生成的。
- 每张人物/场景卡上的“生成此设定图”语义不清，需要变成基于当前可见提示词的局部生成能力。

## 2. 设计目标

- 在生图前先让用户确认人物/场景清单。
- 允许用户新增、编辑、移除和恢复人物/场景项。
- 每个人物/场景都有可见、可编辑的生成提示词。
- 提示词由标准剧本上下文、该人物/场景在剧本里的信息、已确认视觉风格共同生成。
- 每次生成默认 1 张，用户可以调整生成数量。
- 人物默认比例为 `3:4`，场景默认比例为 `16:9`，用户可在生成前修改比例。
- 新生成图片追加到当前卡片的横向候选图池，不覆盖已有候选图。
- 所有人物和场景都选择最终采用图后，才能提交甲方审核。

## 3. 非目标

- 不在本次设计中重做完整脚本导入、格式检查、剧本标准化和文字分镜拆解逻辑。
- 不做独立素材资产管理后台。
- 不做复杂版本树。候选图需要保留生成记录和采用状态，但不需要做多分支版本管理。
- 不把移除项物理删除，避免误删后无法恢复。

## 4. 主流程

1. 用户导入完整剧本，点击“检查并整理”。
2. 剧本格式确认后，AI 拆解详细文字分镜。
3. AI 从标准剧本和文字分镜中抽取人物/场景清单。
4. 进入“清单确认区”，用户确认哪些人物和场景需要生成设定图。
5. 用户可新增、编辑、移除或恢复清单项。
6. 用户确认清单后，进入“设定图生成区”。
7. 系统为每个人物/场景生成初始提示词。
8. 用户在卡片内查看并按需修改提示词、生成数量和比例。
9. 用户点击“按当前提示词生成”，系统只为当前卡片生成新候选图。
10. 新候选图追加到该卡片横向候选图池。
11. 用户为每个人物/场景选择一张最终采用图。
12. 所有必需人物和场景都有采用图后，允许提交甲方审核。
13. 甲方审核通过后，进入下一步分镜图片生产。

## 5. 清单确认区

清单确认区分为人物和场景两个列表。

人物项字段：

- 名称
- 简短描述
- 来源依据，可显示来自哪一场或哪些分镜
- 状态：待确认、已确认、已忽略

场景项字段：

- 名称
- 时间/内外景信息
- 简短描述
- 来源依据，可显示来自哪一场或哪些分镜
- 状态：待确认、已确认、已忽略

可执行操作：

- 新增人物
- 新增场景
- 编辑名称和描述
- 移除到忽略列表
- 从忽略列表恢复
- 确认清单

移除规则：

- “移除”只改变状态为已忽略，不物理删除。
- 已忽略项不参与设定图生成，也不阻塞提交甲方审核。
- 已忽略项保留在折叠的忽略列表中，用户可恢复。

## 6. 设定图生成区 UI

人物和场景都使用横向大卡片，避免竖向长页面堆叠过多信息。

每张卡片包含：

- 左侧基础信息：名称、类型、简短描述。
- 提示词区：可编辑文本框，展示当前用于生成的提示词。
- 控制区：生成数量、比例、生成按钮。
- 候选图池：横向滚动图片列表。

按钮文案：

- 原“生成此设定图”改为“按当前提示词生成”。
- 已有候选图后可以继续使用同一个按钮补图，不需要单独做“补图”按钮。

候选图池规则：

- 初始为空时显示自然语言空状态。
- 每次生成的新图追加到末尾。
- 默认可视区域展示约 4 张候选图。
- 超过 4 张后横向滚动。
- 每张候选图可执行“设为采用”。
- 已采用图片需要有明确选中态。

## 7. 提示词规则

初始提示词由 AI 生成，输入来源包括：

- 标准剧本全文
- 详细文字分镜
- 当前人物/场景的名称和描述
- 该人物/场景出现的场次和上下文
- SOP3 已确认的视觉风格、氛围图或创意方向

提示词必须展示给用户，允许用户直接编辑。

生成时必须使用卡片内当前可见提示词，不允许使用隐藏提示词覆盖用户修改。

每张生成图片需要保存：

- 实际使用的提示词
- 生成比例
- 生成数量批次
- 模型名
- provider
- 任务状态
- OSS 地址
- 失败原因，如有

## 8. 生成数量与比例

生成数量：

- 默认值为 1。
- 用户可在当前卡片内调整。
- 每次点击生成，只为当前人物/场景生成所选数量。
- 新图逐张追加到候选图池。

比例：

- 人物默认 `3:4`。
- 场景默认 `16:9`。
- 用户可在卡片内修改比例。
- 比例随每张生成结果保存，用于后续追溯。

## 9. 进入下一步的门槛

提交甲方审核前必须满足：

- 人物清单已确认。
- 场景清单已确认。
- 所有非忽略人物至少有一张采用图。
- 所有非忽略场景至少有一张采用图。
- 失败或生成中的任务不会被当成采用图。

甲方审核通过后，下一步分镜图片生产必须能自动读取：

- 每条文字分镜中出现的人物。
- 每条文字分镜对应的场景。
- 对应人物和场景的最终采用图。

这些采用图会作为分镜图片生成时的参考图自动注入。

## 10. 数据与 API 影响

需要持久化的核心数据：

- 人物/场景实体状态：待确认、已确认、已忽略。
- 用户编辑后的名称和描述。
- AI 生成的初始提示词。
- 用户编辑后的当前提示词。
- 每次生成时使用的提示词和比例。
- 候选图记录。
- 最终采用图。

API 需要支持：

- 获取人物/场景清单和候选图。
- 新增人物/场景。
- 编辑人物/场景。
- 忽略和恢复人物/场景。
- 确认人物/场景清单。
- 更新卡片提示词。
- 按当前提示词、数量和比例生成候选图。
- 设置最终采用图。
- 提交甲方审核。

所有操作必须写入数据库，刷新页面后能恢复。

## 11. 状态与错误反馈

所有异步动作都需要自然语言反馈：

- 清单抽取中
- 清单抽取失败
- 提示词生成中
- 图片生成排队中
- 图片生成中
- 图片生成失败
- 图片上传 OSS 失败
- 采用图未选齐
- 提交甲方审核成功或失败

失败时需要给出可恢复动作，例如重试生成、修改提示词后再生成、重新上传、重新提交审核。

## 12. 验收标准

- 用户可以在清单确认区新增、编辑、忽略和恢复人物/场景。
- 忽略的人物/场景不参与生图，也不阻塞提交审核。
- 每个人物/场景卡片都能看到并编辑提示词。
- 人物卡默认比例为 `3:4`，场景卡默认比例为 `16:9`。
- 生成数量默认 1，用户修改后只影响当前卡片本次生成。
- 每次生成图片会追加到当前卡片候选图池。
- 候选图超过 4 张后可以横向滚动查看。
- 用户能为每个人物/场景选择最终采用图。
- 采用图未选齐时不能提交甲方审核，并有清晰提示。
- 页面刷新后，清单、提示词、候选图、采用图和任务状态都能恢复。
