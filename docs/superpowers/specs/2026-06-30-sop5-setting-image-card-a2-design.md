# SOP5 设定图生成卡片 A2 设计规格

## 背景

SOP5 的人物/场景设定图生成区当前存在候选图卡片与 Prompt、保存、生成按钮重叠的问题。直接把候选图挪到下方虽然可以止血，但会让卡片变长、变散，不符合生产工具需要的清晰与高密度。

本次确认采用 A2 方案：主图优先的大候选画布。

## 设计目标

- 一张人物或场景卡片内完成对象查看、提示词编辑、参数选择、生成、候选浏览、采用确认。
- AI 在清单确认后，根据完整剧本、人物/场景上下文、关联分镜和已确认视觉风格，自动生成每张卡片的初版 Prompt。
- Prompt 必须显示在卡片文本框中，用户可检查和修改；生成时读取当前文本框内容，不使用后台隐藏 Prompt。
- 中间区域不再横向铺满“图 1、图 2、图 3、图 4、图 5”小卡片。
- 候选图在同一个大画布里左右滑动切换。
- 双击当前候选图可以放大查看。
- Prompt、生成数量、比例、保存、生成、设为采用固定在右侧，不被候选图挤压。
- 每张人物/场景卡片独立生成；点击某张卡片的生成按钮，只为该卡片创建任务，不批量生成全部人物或场景。
- 卡片在常见桌面工作区宽度下不重叠、不横向撑破页面。

## 卡片结构

每张人物/场景设定图卡片分为三块：

1. 左侧对象信息区
   - 展示实体类型：人物或场景。
   - 展示采用状态：待采用或已采用。
   - 展示人物/场景名称。
   - 展示简短描述或来源分镜数量。
   - 展示基础/完整提示词切换。
   - 展示当前参考集版本，例如：基础设定 · v2。

2. 中间候选画布区
   - 只显示一个大候选展示区域。
   - 候选图以横向 scroll-snap 的方式在画布内部切换。
   - 画布顶部显示当前位置，例如：候选 1 / 5。
   - 画布顶部显示提示：双击放大查看。
   - 画布底部显示分页圆点，反馈当前候选位置。
   - 当前候选可以通过双击打开放大弹层。

3. 右侧生成控制区
   - Prompt 文本框，默认由 AI 根据剧本上下文和视觉风格生成，允许用户手动修改。
   - 生成数量选择。
   - 比例选择。
   - 保存按钮。
   - 生成按钮。
   - 设为采用按钮。

## 交互规则

- 确认人物/场景清单后，系统为每张卡片生成初版 Prompt；人物 Prompt 应结合角色年龄、身份、性格、服装、情绪、关键动作和剧本状态，场景 Prompt 应结合地点、时间、内/外景、氛围、道具、光线和关联分镜。
- 点击或横向滑动候选画布，可切换当前候选图。
- 双击当前候选图，打开放大查看弹层。
- 放大查看弹层中应支持：
  - 查看当前候选大图。
  - 左右切换候选。
  - 将当前候选设为采用。
  - 关闭弹层返回卡片。
- 点击“设为采用”后，调用现有 `selectProductionReferenceImage` 能力，并刷新当前工作区数据。
- 点击“保存”后，调用现有 `saveProductionReferencePrompt` 能力。
- 点击“生成”后，调用现有 `generateProductionReferenceImages` 能力，并传入当前卡片的 `entityId`、当前文本框里的 Prompt、生成数量和比例。
- “生成”只作用于当前人物或场景卡片，不触发其他卡片的任务。
- 生成新图片后，图片追加到候选列表中，不覆盖旧候选。
- 人物默认比例为 3:4，场景默认比例为 16:9，仍允许用户修改比例。

## 状态规则

中间候选画布应承载以下状态：

- 空状态：还没有候选图，提示用户先确认提示词并生成。
- 排队中：显示“排队中”，不允许设为采用。
- 生成中：显示“生成中”，不允许设为采用。
- 生成失败：显示自然语言失败原因和可重试提示。
- 已生成：展示图片，可设为采用。
- 已采用：画布和状态区都明确显示已采用。

右侧按钮状态：

- 当前实体已锁定时，Prompt、参数、保存、生成、设为采用均禁用。
- 当前图片不是成功状态时，设为采用禁用。
- 生成任务进行中时，生成按钮显示 loading，并避免重复提交。
- Prompt 为空时，生成按钮应返回自然语言错误，提示先填写当前卡片提示词。

## 响应式规则

- 宽桌面：三列布局，左侧对象信息，中间候选画布，右侧控制区。
- 中等宽度：左侧对象信息 + 中间候选画布保持同一行，右侧控制区下移成整行。
- 窄宽度：三块纵向排列。
- 不允许候选图区域覆盖 Prompt、按钮或下一张实体卡片。
- 不允许整张卡片产生页面级横向滚动。

## 测试与验收

- 源码测试应断言 SOP5 设定图生成区存在：
  - 主候选画布文案或类名。
  - 双击放大查看入口。
  - Prompt、生成数量、比例、保存、生成、设为采用。
  - `generateProductionReferenceImages` 调用使用当前卡片 `entityId` 和当前卡片 Prompt。
  - 不再出现“按当前提示词生成”和“生成此设定图”。
  - 不再使用旧的三列小卡片候选池作为主要展示。
- 浏览器验收应覆盖：
  - 有 0 张候选图时不卡住、不重叠。
  - 有 1 张候选图时主画布展示正常。
  - 有 5 张及以上候选图时可在画布内部横向滑动。
  - 双击候选图能打开放大弹层。
  - 1280px 左右桌面宽度下不重叠。
  - 窄屏下卡片纵向排列但内容仍完整。

## 不做范围

- 不改变人物/场景实体的数据结构。
- 不改变生成图片、保存 Prompt、选择采用图的后端接口。
- 不重做 SOP5 全流程。
- 不引入新的图片生成 provider。
- 不把候选图改成静态 mock 数据。
- 不增加全局批量生成人物/场景设定图入口。
