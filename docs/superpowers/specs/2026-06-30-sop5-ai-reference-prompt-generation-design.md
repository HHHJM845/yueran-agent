# SOP5 AI 设定图 Prompt 生成设计规格

## 背景

SOP5 人物/场景设定图生成区已经具备 Prompt 输入框、生成数量、比例、生成按钮和候选图画布。但当前 Prompt 的来源主要是后端本地模板拼接，不能充分理解完整剧本、文字分镜、人物关系、场景用途和已确认视觉风格。

正确逻辑应是：AI 在人物/场景清单确认后，为每个有效人物和场景自动生成可编辑的设定图 Prompt，并填入卡片右侧 Prompt 框。用户可修改后再正式生图。

## 设计目标

- 人物/场景清单确认前，不自动生成最终设定图 Prompt。
- 用户确认清单后，系统基于完整剧本、文字分镜和已确认风格，自动生成每个有效人物/场景的 Prompt。
- 生成出的 Prompt 必须直接显示在 A2 卡片右侧 Prompt 框内。
- 用户可以继续手动编辑 Prompt，并保存覆盖。
- 保留模板 Prompt 作为兜底，但不可作为主要生成结果。
- 人物默认比例为 3:4，场景默认比例为 16:9。
- 后续生图必须使用用户当前可见、可编辑的 Prompt。

## 流程设计

### 1. 自动拆分文字分镜

系统拆分文字分镜后，仍按现有逻辑抽取人物和场景清单，并创建对应 reference set。

此时 reference set 可以有兜底 Prompt，但 UI 应提示这不是最终 AI 设定 Prompt。

### 2. 清单编辑

用户在“清单确认区”完成：

- 新增漏识别的人物或场景。
- 编辑人物或场景名称。
- 编辑人物或场景简短描述。
- 将路人、群众、无关对象移入忽略列表。
- 恢复误忽略的人物或场景。

### 3. 确认清单并生成 Prompt

用户点击“确认清单”后，系统执行两个动作：

1. 将有效人物/场景标记为已确认。
2. 调用 AI 为所有有效人物/场景生成设定图 Prompt。

Prompt 生成完成后，写入对应 `production_reference_sets.current_prompt` 和 `prompt` 字段，A2 卡片右侧 Prompt 框自动显示最新内容。

### 4. 重新生成 Prompt

清单确认后，设定图生成区提供“重新生成提示词”按钮。

使用场景：

- 用户调整了人物/场景描述。
- 用户修改了创意方向或视觉风格。
- 用户觉得 Prompt 不够准确。
- 手动新增了人物或场景。

重新生成时只覆盖未锁定的 reference set。已锁定或已通过甲方审核的人物/场景不自动覆盖，避免破坏已确认资产。

## AI 输入上下文

AI 生成 Prompt 时必须读取以下信息：

- 项目基础信息：品牌名、项目名。
- 已确认创意方向：标题、核心概念、视觉风格、参考标签、氛围图 Prompt。
- 完整剧本：剧名、剧情简介、人物小传、正式剧本内容。
- 文字分镜：场次、镜号、画面描述、动作、人物引用、场景引用。
- 当前人物/场景清单：类型、名称、描述、重要程度、来源分镜 ID。
- 每个人物/场景关联的上下文：
  - 出现在哪些场次和分镜。
  - 在这些分镜中的动作、状态、情绪或空间功能。
  - 与其他人物或场景的关系。

## AI 输出结构

AI 必须输出严格 JSON，不允许 Markdown。

结构如下：

```json
{
  "items": [
    {
      "entityId": "production_entity_id",
      "entityType": "character",
      "name": "阿辰",
      "basicPrompt": "用于快速生成人物基础设定图的 Prompt",
      "fullPrompt": "用于完整人物设定图的 Prompt",
      "defaultRatio": "3:4",
      "reasoningSummary": "简短说明 Prompt 如何来自剧本、分镜和风格"
    }
  ]
}
```

字段规则：

- `entityId` 必须匹配现有 production entity。
- `entityType` 只能是 `character`、`scene` 或 `prop`。
- `basicPrompt` 用于基础设定，必须清晰描述主体、关键外观、情绪、服装/空间、风格。
- `fullPrompt` 用于完整设定，必须在 basicPrompt 基础上补充细节、材质、光线、构图、可复用参考要求。
- `defaultRatio` 人物默认 `3:4`，场景默认 `16:9`，道具默认 `1:1`。
- `reasoningSummary` 用于后端快照和审计，不直接占据主 UI。

## Prompt 内容要求

人物 Prompt 应包含：

- 人物身份和年龄气质。
- 剧本中的关键动作或情绪。
- 服装、道具、体态、表情。
- 所属项目的视觉风格。
- 背景可以轻描淡写，但不能喧宾夺主。
- 禁止生成多个不一致人物。
- 禁止出现无关品牌、无关文字、水印、畸形肢体。

场景 Prompt 应包含：

- 场景地点、时间、内外景。
- 空间结构和关键视觉元素。
- 与剧本动作相关的布景重点。
- 光线、氛围、镜头感。
- 所属项目的视觉风格。
- 禁止无关人物抢主体。
- 禁止出现无关文字、水印、错乱透视。

## UI 行为

### 清单确认区

- 按钮文案保持“确认清单”。
- 点击后显示自然语言 loading：正在根据剧本和已确认风格生成设定图提示词。
- 成功后显示：人物和场景清单已确认，设定图提示词已生成。
- 失败后显示：清单已确认，但提示词生成失败；可点击“重新生成提示词”。

### 设定图生成区

每张 A2 卡片右侧 Prompt 框展示：

- 已生成 Prompt：直接展示可编辑 Prompt。
- 未生成 Prompt：显示“请先确认清单，系统会根据剧本和视觉风格生成提示词。”
- 生成中：Prompt 框禁用并显示生成中提示。
- 生成失败：保留旧 Prompt 或兜底 Prompt，并显示错误提示。

设定图生成区顶部增加：

- “重新生成提示词”按钮。
- 显示最近一次 Prompt 生成状态：未生成、生成中、已生成、生成失败。

## 数据与持久化

- 不新增前端内存型状态保存 Prompt。
- Prompt 生成结果必须写入数据库 `production_reference_sets.prompt` 和 `production_reference_sets.current_prompt`。
- `snapshot_json` 中记录：
  - promptSource: `ai_script_context`
  - sourcePackageId
  - sourceCreativeDirectionIds
  - sourceShotIds
  - styleSummary
  - reasoningSummary
  - generatedAt
- 用户手动保存 Prompt 后，仍使用现有保存能力，并可将 `snapshot_json.promptSource` 更新为 `manual_edit`。

## 错误处理

- 如果没有完整剧本，提示：请先导入完整剧本并完成格式检查。
- 如果没有文字分镜，提示：请先拆分文字分镜。
- 如果没有有效人物/场景，提示：请先确认人物和场景清单。
- 如果 AI provider 未配置，提示：文本模型未配置，暂时不能生成提示词。
- 如果 AI 输出结构不完整，提示：AI 已返回内容但结构不完整，请重试或手动填写 Prompt。
- 所有错误都必须保存到可诊断日志或阶段状态，不直接暴露模型原始异常。

## 不做范围

- 不改变 OpenAI 生图 provider。
- 不改变图片候选画布 A2 布局。
- 不改变分镜图片 Prompt 生成逻辑。
- 不改变视频 Prompt 生成逻辑。
- 不自动覆盖已锁定或已通过甲方审核的设定图 Prompt。
- 不把 AI Prompt 生成结果做成不可编辑黑盒。

## 验收标准

- 点击“确认清单”后，会自动触发 AI Prompt 生成。
- 生成成功后，每个人物/场景卡片右侧 Prompt 框都有具体内容。
- Prompt 内容能体现剧本上下文、人物/场景用途和已确认视觉风格。
- 手动编辑 Prompt 后保存，刷新页面仍能看到修改后的 Prompt。
- 点击“按当前提示词生成”时，使用的是当前框内可见 Prompt。
- 生成失败时，用户能看到自然语言错误，并可重新生成。
- 已锁定或已通过甲方审核的 reference set 不被重新生成覆盖。
