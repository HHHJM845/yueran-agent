# SOP 3 聚焦式创意视觉提案设计

日期：2026-06-30

## 背景

当前 SOP 3 `creative_direction_proposal` 工作区把资料解析、四个创意方向、故事大纲、氛围图、Round 1/2 甲方审核和最终提案整理同时堆在页面里。用户需要自己判断“现在该做哪一步”，页面主体也会把过往阶段和未来阶段一起展示出来。

本次设计目标是延续 SOP 1 / SOP 2 的简洁原则：工作台主体永远聚焦当下任务，后台仍保留完整业务流转、历史快照和真实状态机。

## 目标

把 SOP 3 改成“当前任务面板 + 底部小流程进展图”的体验：

- 系统根据现有业务数据判断当前 SOP 3 子步骤。
- 页面标题和主按钮使用人工任务语言，例如“选择要发给甲方的方向”“等待甲方初筛”“深化已确认方向”。
- 主工作区只展示当前步骤需要操作或判断的内容。
- 过往记录、Round 快照、甲方反馈和后续预览放到底部小流程进展图里点击查看。
- 后端生成、甲方审核、氛围图任务和 SOP 4 准入规则不改。

## 非目标

- 不新增完整 SOP 3 数据库子状态机。
- 不实现回滚能力；底部进展图只做历史预览，不改变项目状态。
- 不修改甲方审核链接、验证码、提交回写的后端逻辑。
- 不删除已持久化的 `creative_directions`、`creative_expansions`、`generated_images`、`creative_proposal_rounds` 数据。
- 不把 SOP 4 报价合同逻辑并入 SOP 3。
- 不常驻展示资料解析结果、Round 1/2 详情、完整提案编辑器等非当前任务内容。

## 当前任务判断

新增前端 view-model，用现有工作区数据派生当前任务，不新增字段。

建议状态顺序：

1. `generate_directions`
   - 条件：没有创意方向，或方向生成任务失败后需要重试。
   - 主体：空状态、生成 4 个方向按钮、自然语言错误或 loading。

2. `select_directions`
   - 条件：已有 4 个方向，但没有匹配当前选择的 Round 1。
   - 主体：只显示 4 张方向卡，支持内部单选或多选；主按钮是“发送给甲方初筛”。

3. `wait_round_1_feedback`
   - 条件：Round 1 已创建或已发起甲方审核，但还没有甲方有效回传。
   - 主体：显示本轮发送状态、审核链接或历史发送信息、刷新提示；不显示深化内容。

4. `deepen_confirmed_direction`
   - 条件：Round 1 有甲方反馈，且系统已经得到需要深化的方向；Round 2 尚未完成。
   - 主体：只显示甲方确认或保留的方向，生成故事大纲和氛围图；未选方向收进历史预览。

5. `wait_round_2_feedback`
   - 条件：Round 2 已创建或已发起甲方审核，但最终确认尚未回传。
   - 主体：显示 Round 2 发送状态、审核链接或历史发送信息、刷新提示。

6. `finalize_proposal`
   - 条件：Round 2 已通过或最终创意方向已确认。
   - 主体：显示已确认方向摘要、已确认故事大纲/氛围图，提供进入最终提案整理或 SOP 4 的主操作。

如果数据处于不完整状态，例如方向不足 4 个、Round 快照与当前选择不一致、甲方反馈缺失关键字段，当前任务面板显示自然语言修复建议，不静默失败。

## 主工作区设计

### 顶部

- 标题显示当前任务，不显示笼统流程名。
- 副文案只解释当前要做什么。
- 右侧最多显示 1-2 个状态 pill，例如“等待甲方”“已选 2 个方向”。
- 不展示 `SOP 3`、`Round 1/2 全流程说明`、`四个方向/两轮提案/甲方反馈` 这类总览装饰。

### 当前任务主体

不同任务只渲染自己的主体组件：

- 生成方向：一个生成按钮和任务状态。
- 选择方向：四个方向卡片，卡片内保留标题、核心想法、适配理由、风险提示和选择状态；编辑能力可收进卡片操作。
- 等待甲方：一个等待面板，展示本轮甲方审核状态、链接/验证码展示规则、已提交反馈摘要或刷新提示。
- 深化方向：只展示已确认方向的故事大纲、氛围图生成和采用/废弃操作。
- 最终确认：展示最终方向与产物摘要，提供进入提案整理或下一阶段的操作。

不属于当前任务的模块不在主体常驻展示。

## 底部流程进展图

页面底部保留一个小型流程进展图，默认不抢占视觉重心。

节点建议：

1. 方向生成
2. 内部选择
3. 甲方初筛
4. 方向深化
5. 最终确认

每个节点显示：

- 状态：未开始、当前、已完成、需要处理。
- 简短摘要：例如“已生成 4 个方向”“甲方已选择 2 个方向”“等待回传”。

点击已完成节点时，在原地展开历史预览：

- 方向生成：展示四方向标题和生成时间。
- 内部选择：展示当时选择了哪些方向。
- 甲方初筛：展示 Round 1 状态、甲方选择和备注摘要。
- 方向深化：展示已生成故事大纲数量、氛围图数量和采用情况。
- 最终确认：展示 Round 2 状态和最终方向摘要。

历史预览只读，不改变主体工作区，不触发回滚。

## 数据流

1. 页面加载现有 `WorkspaceData`：
   - `creativeDirections`
   - `creativeExpansions`
   - `generatedImages`
   - `creativeProposalRounds`
   - `clientReviewTasks`
   - `jobs`
   - `artifacts`

2. 前端 view-model 派生：
   - `currentTask`
   - `primaryAction`
   - `visibleDirections`
   - `visibleExpansions`
   - `progressNodes`
   - `historyPreview`
   - `blockingMessage`

3. 用户操作仍调用现有 API：
   - `generateCreativeDirections`
   - `updateCreativeDirectionSelection`
   - `createCreativeProposalRound`
   - `createCreativeProposalRoundClientReview`
   - `generateCreativeExpansions`
   - `generateAtmosphereImage`
   - `selectCreativeSceneImages`

4. 后端完成任务或甲方回传后，前端刷新工作区，view-model 自动切换当前任务。

## 权限

- 创意团队和管理员可以生成方向、编辑方向、生成深化内容和氛围图。
- 商务团队可以选择方向并发起甲方审核。
- 甲方审核链接仍由现有 client review 流程处理。
- 无权限用户可以查看当前任务和历史预览，但不能提交操作。

## 错误与空状态

所有异常必须是自然语言：

- 没有方向：提示“还没有创意方向，请先生成 4 个方向。”
- 方向不足 4 个：提示“当前方向数量不完整，请重新生成后再发给甲方。”
- 未选择方向：提示“请至少选择 1 个方向，再发送给甲方初筛。”
- 甲方反馈缺失：提示“还没有收到甲方初筛结果，可以稍后刷新或重新发送审核链接。”
- 生成失败：沿用 job 的 `userMessage`，不展示原始模型或 SDK 错误。

## 测试与基线

需要新增或扩展回归测试，保护以下行为：

- SOP 3 工作区不再同时常驻显示资料解析、四方向、深化板、Round 面板和最终提案整理。
- 当前任务为“选择方向”时，只显示四个方向和发送甲方主操作。
- 当前任务为“等待甲方初筛”时，不显示故事大纲和氛围图生成区。
- 当前任务为“深化已确认方向”时，只显示已确认方向，不把未选方向作为主体内容展示。
- 底部进展图存在，点击历史节点只展开只读预览，不改变当前任务。
- `createCreativeProposalRound`、`createCreativeProposalRoundClientReview`、`generateCreativeExpansions` 等后端调用契约不变。
- `npm run test:baseline` 继续通过；如 SOP3 加入基线，需把新测试纳入该命令。

验证命令：

```bash
npm run test:baseline
npm run typecheck
npm run lint
npm run build
```

涉及 UI 实现后，需要浏览器检查：

- 无方向空状态。
- 四方向选择态。
- 等待甲方回传态。
- 已确认方向深化态。
- 底部流程进展图历史预览。

## 验收标准

- 用户进入 SOP 3 时，一眼能知道当前该做什么。
- 主工作区只处理当下任务，不堆叠历史阶段和未来阶段。
- 四个方向生成后，下方只显示四个方向。
- 内部选择后，可以直接发送给甲方初筛。
- 甲方回传后，工作区自动切到下一步或深化已确认方向。
- 深化时只展示当前要深化的方向和内容。
- 底部进展图可以查看过往记录，但不会打断当前工作。
- 后端状态机、甲方审核、job worker 和 SOP 4 准入规则保持一致。
